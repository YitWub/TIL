# 기초 문법

##  반복
- **String.join** : 
```java
String.join(구분자, 문자열배열) 
ex > String.join("", arr);
```
ㄴ 구분자 ""이라 공백없이 붙여짐

- 장점: 
String.join은 내부적으로 자원을 효율적으로 관리하는 StringJoiner를 사용. 합쳐질 문자열들의 전체 길이를 미리 계산하고 필요한 만큼만 메모리를 한 번에 할당하여 문자열을 조합하기 때문에, 반복적인 객체 생성과 메모리 복사 비용이 발생하지 않음

- **for-each** :
for (타입 변수명 : 배열 혹은 컬렉션)

- ex:
```java
for(String a : arr) {
answer += a;
}
```
a 가 변수가 되어서 for문 돌면서 요소 answer에 넣어줌

- 단점: String 객체는 한 번 생성되면 그 값을 변경x
 따라서 반복문 안에서 answer += arr[i]를 실행하면, 기존 answer 문자열에 새로운 문자열을 덧붙이는 것이 아닌  매번 기존 문자열과 새로운 문자열을 읽어 메모리에 완전히 새로운 String 객체를 할당하게 됨. -> 메모리, 시간복잡도 증가

## 배열

- **선언**
int[] answer = {}; (빈 배열 초기화)
 ㄴ 길이가 0인 배열객체가 메모리의 HEAP영역에 생성
 ㄴ answer라는 변수가 방금 만든 빈 배열 객체의 메모리 주소를 정상적으로 가리키고 있는 상태

int[] answer; (비교군)
 ㄴ 메모리에 아무런 객체도 생성 안됨. 배열을 가르킬수있는 껍데기 이름표(참조변수)만 만들음
 ㄴ 사용하면 컴파일에러
